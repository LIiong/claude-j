# 开发日志 — 007-shopping-cart

## 问题记录

### 1. 购物车金额值对象复用问题
- **问题**：order 聚合中已有 Money 值对象，cart 聚合是否应直接引用
- **决策**：在 cart 聚合下创建独立的 Money 值对象，与 order 的 Money 物理隔离
- **原因**：DDD 要求聚合边界独立，跨聚合引用 domain 内部值对象会造成耦合。若未来 order 的 Money 变更（如增加汇率转换），不应影响 cart 聚合。两者逻辑相同但物理隔离，后续可考虑提取到 common 包

### 2. 购物车聚合粒度
- **问题**：购物车是独立聚合还是作为 user 聚合的一部分
- **决策**：购物车作为独立聚合（Cart），通过 userId 关联用户
- **原因**：购物车有独立的生命周期和业务规则（增删改查商品、金额计算），与 user 聚合的关注点不同。独立聚合便于独立变更和扩展

### 3. 相同商品合并策略
- **问题**：用户添加已在购物车中的商品时如何处理
- **决策**：累加数量到已有购物车项，而非新增记录
- **原因**：符合电商购物车主流 UX 模式，减少重复项，简化用户操作
