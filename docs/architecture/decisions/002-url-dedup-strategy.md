# ADR-002: URL 去重策略选择 SHA-256 哈希索引

**日期**：2026-03-24
**状态**：已接受
**决策者**：@dev

## 背景
相同 URL 多次提交应返回同一短码（去重）。原始 URL 最长 2048 字符，直接在 VARCHAR(2048) 上建索引不现实。

## 决策
在 `t_short_link` 表增加 `original_url_hash VARCHAR(64)` 列，存储 URL 的 SHA-256 哈希值，在该列建普通索引 `idx_original_url_hash`。查询时先按哈希匹配，再比对原始 URL 确认。

## 备选方案
| 方案 | 优点 | 缺点 | 排除原因 |
|------|------|------|----------|
| 直接索引 VARCHAR(2048) | 简单 | MySQL 索引长度限制 767 字节 | 技术不可行 |
| 前缀索引 VARCHAR(255) | 可行 | 前缀相同 URL 区分度不够 | 千万级下误匹配率高 |
| CRC32 哈希 | 计算快 | 32位碰撞概率高 | 千万级下不安全 |

## 影响
- SHA-256 碰撞概率可忽略（2^128 级别安全性）
- 每条记录额外存储 64 字符哈希值
- Repository 实现层负责计算哈希，domain 层不感知
