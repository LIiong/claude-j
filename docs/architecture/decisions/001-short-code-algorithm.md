# ADR-001: 短码生成算法选择 Base62(ID) + 乘法逆元位混淆

**日期**：2026-03-24
**状态**：已接受
**决策者**：用户 + @dev

## 背景
短链服务需要根据长 URL 生成 6 位短码，要求：
- 千万级数据量零碰撞
- 短码不可预测（不能通过短码推算 ID 顺序）
- 算法复杂度适中

## 决策
采用 **Base62(自增ID) + 乘法逆元位混淆** 方案。
- 先 insert 获取自增 ID
- 用 `(id * PRIME) & (MAX - 1)` 进行位混淆（PRIME = 2654435761L）
- 对混淆值进行 Base62 编码，左填充到 6 位
- 回写 short_code 到记录

## 备选方案
| 方案 | 优点 | 缺点 | 排除原因 |
|------|------|------|----------|
| 纯 Base62(自增ID) | 零碰撞，简单 | 可预测：id=1→"1", id=2→"2" | 安全性不足 |
| 随机生成 + 碰撞重试 | 完全随机 | 高并发碰撞率高，需重试 | 千万级下碰撞概率不可忽略 |
| MD5/SHA 截断 | 不依赖 ID | 碰撞概率高，需处理冲突 | 不确定性高 |
| Snowflake | 分布式唯一 | 生成的 ID 过长，不适合短码 | 码长超 6 位 |

## 影响
- 短码与自增 ID 存在确定性映射关系（可逆），但外部无法推算
- 6 位 Base62 = 62^6 ≈ 568 亿种组合，远超千万级需求
- 需要两次 save（insert 获取 ID → update 写入 short_code）
